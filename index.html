Frequently Asked Questions

What is the difference between var, let, and const?
var is function scoped. We can reassign a new value to a variable which is created by var. Hoisting occures if we create a variable with var. let is a block scoped. We can reassign a new value to a variebale which is created with let. Hoisting is not possible in let. const is block scoped. It is also read only this means we cannot change the value of a variable which is created with const. Hoisting is not possible in const.
The difference between map(), forEach(), and filter()
All of these methods can be used with array. These are array methods. We use map() when we need a new array in return. map() method will return something. forEach() does not return anything. We use forEach() method to simply iterate over the array elemnets. filter() method returns a new array with filterd elemnets from the previous array. We use this method to filter out unwanted array elemnets.
Explain arrow functions and how they are different from regular functions
The syntax of an arrow function is different from the regular functions. arrow functions uses shorter syntax for example: (params) => expression or (params) => { statements } whereas regular functions have bigger syntax for example: function name(params) { statements }. Arrow functions do not have their own argument object but regular functions have arguments object. Arrow functions inherit the "this" keyword from the surrounding scope but the regular function interits "this" keyword form the global scope.
How JavaScript Promises work?
The JvaScript promise works in three ways, they are: Pending,Fulfilled or rejected. If the promise shows pending this means it is in its initial phase not fulfilled nor rejected. if the promise shows fulfilled this means that the operations completed succesfully. If the promise shows rejected this means that the operation is failed and it will thorw an error. Promise provide a structured way to handle asynchronous codes.
How closures work in JavaScript?
Clouser in JavaScript allows a function to access variables from its outer scopes even after the outer function has finished its operations. Clouser remembers the environment in which it was created allowing them to access the variables form that environment later.
